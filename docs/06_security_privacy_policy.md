# 06. セキュリティ・プライバシー・免責事項の方針

## 方針サマリー

| 項目 | 方針 |
|---|---|
| 医療性 | 診断・治療・投薬量助言・飲み合わせ判定を**一切実装しない**。用量は獣医指示の自由文をそのまま記録 |
| 免責表示 | オンボーディング初回画面・LPフッター・プライバシーポリシーの3か所に「医療機器ではない」旨を常設 |
| 個人情報 | 氏名・住所・電話番号を取得しない。認証は匿名。メールはLP事前登録の任意入力のみ |
| 分析イベント | ペット名・薬名・用量・メモ等の入力値を送信しない（docs/07 規約） |
| データ削除 | アプリ内導線（設定→退出/世帯全削除）+ Web導線（privacy.htmlの連絡先）。Play「アカウント削除」要件対応 |
| 通信 | Firebase SDK経由のTLSのみ。独自エンドポイントなし |
| アクセス制御 | Firestore rules: 世帯メンバーのみ read/write、doseRecords append-only、invites list禁止（android/firestore.rules） |
| 通知不達の免責 | プライバシーポリシー6条で「通知の不達を含む損害への免責」を明示（端末省電力による不達は完全には防げない） |

## Google Play 提出時の必須対応（ユーザー作業を含む）

1. **健康アプリ申告**: Play Console「アプリのコンテンツ」→ Health apps 申告。カテゴリは
   「健康・フィットネス（ペット向け記録）」として申告し、医療機器・診断機能なしと回答
2. **Data Safety フォーム**（下記の収集内容宣言を転記）
3. **プライバシーポリシーURL**: `https://aomizuki0307.github.io/pet-med-lp/privacy.html`（LP公開後に有効）
4. **アカウント削除URL**: 同ポリシー5条を指すアンカーURLを申告（アプリ内削除導線あり、と申告）

### Data Safety 宣言内容（現行実装ベース）

| データ種別 | 収集 | 共有 | 目的 |
|---|---|---|---|
| アプリのインタラクション（events） | ○ | × | 分析 |
| ユーザー生成コンテンツ（ペット・薬・投薬記録） | ○（クラウド同期） | × | アプリ機能（家族共有） |
| 匿名ID（Firebase Auth uid） | ○ | × | アプリ機能 |
| 位置情報・連絡先・写真等 | × | — | — |

- すべて暗号化転送（TLS）/ 削除リクエスト可能（アプリ内）と申告

## 免責文（正文・アプリ内とLPで共通）

> 本アプリは医療機器ではありません。ペットの診断・治療・投薬量の判断に関する機能は提供せず、
> 記録と共有のみを行います。投薬内容は必ず獣医師の指示に従ってください。
> 本アプリの利用または利用不能（通知の不達を含む）により生じた損害について、開発者は責任を負いかねます。

## 脅威と対応（検証MVPの水準）

| 脅威 | 対応 |
|---|---|
| 他世帯への侵入（rulesバグ） | invites get()検証 + list禁止。公開前にエミュレータで docs/08 チェックリスト必須 |
| 招待コード総当たり | 6桁×72h期限×list禁止。30世帯規模では十分（本実装時はレート制限/コード長を再検討） |
| 記録の改ざん | doseRecords update/delete 拒否（append-only）。訂正はCANCELLED追加で痕跡が残る |
| データ消失 | クラウド保存が一次。端末はFirestoreキャッシュ。閉テスト中にexport/restore試験（docs/08） |
| 分析へのPII混入 | AnalyticsLogger 経由のみ送信可・自由文パラメータなし（コードレビュー時の確認項目） |
| GAS計測エンドポイント荒らし | URLトークン照合（不一致は記録もしない）+ シート数式インジェクション対策 |
