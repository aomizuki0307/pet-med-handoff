# 08. クローズドテスト実施手順

## 前提

- Google Play Developer アカウント保有（確認済み）
- Firebase設定完了（docs/10）+ `assembleProdRelease` 用の署名鍵作成（docs/11）
- プライバシーポリシーURL公開済み（LPのprivacy.html）

## Play Console 手順

1. アプリ作成: 名称「おくすり当番（ペット投薬共有）」/ 日本語 / アプリ / 無料
2. 「アプリのコンテンツ」を全て埋める: プライバシーポリシーURL / 広告なし /
   対象年齢（全年齢・子ども向けでない）/ **健康アプリ申告** / Data Safety（docs/06の表を転記）
3. **内部テスト**トラックにAABをアップロード（まず自分の端末で動作確認）
4. **クローズドテスト**トラック作成 → テスターのメールリスト登録（事前登録者から招待）
   - オプトインURLを事前登録者へメールで送付（LPのGAS Preregisterシートから抽出）
5. 審査提出（クローズドテストも審査あり。健康申告の回答と実装の一致を確認してから提出）

## テスター向け案内文（テンプレ）

> 「おくすり当番」クローズドテストへのご協力ありがとうございます。
> 1. このリンクからテスターに参加 → Playストアからインストール
> 2. 最初にあなたの表示名とペット・お薬を登録してください（2分ほど）
> 3. ご家族がいる場合は、アプリ右上の招待アイコンから6桁コードを共有してください
> 4. 2週間、ふだん通りに投薬記録をつけてみてください
> 5. 動かない・わかりにくい点はこのメールに返信でお知らせください（すべて読みます）
> ※記録内容が医療アドバイスに使われることはありません。投薬は獣医師の指示に従ってください。

## 端末横断・手動テスト表（公開前必須）

| # | 試験 | 手順 | 合格基準 |
|---|---|---|---|
| T1 | 通知権限フロー | 新規インストール→薬登録→権限ダイアログ | 拒否してもクラッシュせず、Todayに警告カード表示 |
| T2 | 正確なアラーム拒否 | 設定でアラーム&リマインダーをOFF | ±10分窓で通知が来る。カードから設定へ遷移できる |
| T3 | 通知1タップ記録 | 通知の「投薬済み」タップ | 記録が作成され、他端末に反映。通知が消える |
| T4 | 再起動復元 | アラームセット後に端末再起動 | 再起動後も通知が発火（BootReceiver） |
| T5 | 時刻変更 | 端末時刻を手動で前後させる | アラームが再計算される |
| T6 | オフライン記録 | 機内モードで記録→解除 | 記録が消えず同期される。重複時はバナー表示 |
| T7 | 二重投薬警告 | 2端末で同一スロットに記録 | 事前警告（オンライン時）or 同期後バナー |
| T8 | 世帯参加 | 招待コードで2台目参加 | 参加後、既存のペット・薬・履歴が見える |
| T9 | 不正参加の拒否 | 期限切れ/でたらめコード | 参加できずエラー表示 |
| T10 | データ削除 | 設定→退出/世帯全削除 | Firestoreからドキュメント消滅（コンソールで確認） |
| T11 | 無料枠制限 | 14日経過を模擬（createdAtを操作） | 2件目の薬・2人目の招待・8日前の履歴がロック |
| T12 | 省電力端末 | 対象端末でバッテリー最適化ON | 通知遅延を記録（alarm_fired delayMinutesで実測） |

## Firestore rules 手動チェックリスト（エミュレータ or 実機2アカウント）

- [ ] 非メンバーuidで他世帯の household/pets/doseRecords が read できない
- [ ] 非メンバーuidで doseRecords が create できない
- [ ] メンバーでも doseRecords の update / delete が拒否される
- [ ] 期限切れinviteコードで members create が拒否される
- [ ] revoked=true のinviteで参加できない
- [ ] invites の list クエリが拒否される
- [ ] events が read できない（create のみ成功）

## 監視（テスト期間中・毎日5分）

- Firestore `events`: sync_error / app_error の件数
- alarm_fired の delayMinutes 分布（±15分KPI）
- 問い合わせ対応時間を分単位で記録（→ docs/12 のサポート集計へ）
