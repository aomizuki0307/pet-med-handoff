# 02. 検証仮説と計測指標

> 30日検証の判断基準。数値はDDレポートE節・H節・依頼文の合格基準に準拠。
> 判定記入先: `12_go_no_go_template.md`

## 仮説（優先順）

| # | 仮説 | 計測方法 | 合格基準 | 中止シグナル |
|---|---|---|---|---|
| H1 | 家族間での投薬状況共有は、実際に繰り返し発生する問題を解決する | 二人目メンバーによる dose_recorded 発生世帯率 / 二重警告表示回数 | 共有世帯の過半で二人目の記録が発生 | 家族招待がほとんど使われない |
| H2 | 投薬完了の記録が週4日以上利用される | 世帯別・週別の記録日数（dose_recorded の distinct slotDate） | 週4日以上記録する世帯 ≥50% | 週4日世帯 <30% |
| H3 | 家族をアプリへ招待する必要性がある | invite_created → invite_accepted 率 | 家族招待率 ≥30% | 招待率 <15% |
| H4 | 二重投薬防止と履歴共有に月400円を支払う意向がある | LP価格CTA / アプリ内 purchase_intent（アクティブ世帯比） | LP登録率≥6% かつ 有料意向≥40% | LP価格付き登録率 <3% / 意向 <20% |
| H5 | 一般的なペット健康管理機能がなくても価値が成立する | 問い合わせ・要望の分類（健康記録要望の比率） | 継続世帯の主要離脱理由が「機能不足」でない | 「食事/体重がないと使えない」が離脱理由の過半 |
| H6 | 問い合わせと運営負担を小さく維持できる | サポート対応時間の記録（分単位） | 月8時間以下 | 月12時間超の見込み |

## 合格基準サマリー（依頼文準拠）

- 価格を表示したLPの登録率: **≥6%**（中止 <3%）
- テスト開始世帯の2週目継続率: **≥70%**（中止 <50%）
- 週4日以上記録する世帯: **≥50%**
- 月400円での利用意向: **≥40%**
- 家族招待率: **≥30%**
- 問い合わせ負担: **月8時間以下**（中止 >12時間見込み）

## その他の中止・方向転換シグナル

- 個人の投薬リマインダーだけで十分と判明（招待率・二人目記録の両方が低い）
- 無料競合（ペットノートPlus等）との差を説明できない
- 医療上の問い合わせ・責任リスクが大きすぎる

## 30日後の意思決定（3択のみ — DDレポートH節）

1. **着手**: CTA≥6% / W2≥70% / アクティブ中有料意向≥40% / サポート≤8h → 本実装（Billing・エクスポート等）へ
2. **1回だけ修正検証**: CTA3〜6% または W2 50〜70% → 訴求を「投薬監査」か「受診PDF」に限定し14日再試験
3. **中止して2位（釣果記録）へ**: CTA<3% / W2<50% / 健康データ運営負担が許容外

## 継続率の集計単位に関する注意

匿名認証のため、アンインストール→再インストールで uid が変わる。
**継続率は uid ではなく householdId 単位**で集計する（events に householdId の短縮ハッシュを併記）。
