# 01. 要件定義（検証用MVP）

> 正典: `AI_coding/docs/research/android_top5_due_diligence_2026-07-17.md` F節（最終推奨案）
> 本書はレポートのMVPスコープを実装可能な要件に落としたもの。**需要検証が目的であり、完成品ではない。**

## 価値提案（1文）

高齢・慢性疾患の犬猫について、家族全員が「誰が・いつ・何を投与したか」を1タップで共有し、
飲ませ忘れと二重投薬を防ぐAndroidアプリ。

## 対象ユーザー

- 高齢または持病（腎臓病・糖尿病・心臓病・術後等）の犬猫を飼っている日本の世帯
- 毎日または定期的な投薬がある
- 2人以上で世話を分担し、「誰が投薬したか分からなくなる」ことがある
- 一般的なペット日記ユーザーは対象外

## 固定条件（変更不可）

- Kotlin + Jetpack Compose / 日本語圏ファースト（将来の英語対応を妨げない: strings.xml集約・ISO日時・TZ考慮）
- セルフサービス型（電話・訪問・商談なし）/ 月5万円の安定利益目標 / 広告依存なし / 継続コンテンツ制作なし
- AI診断・医療助言なし / 本番規模バックエンドなし / 未検証機能を増やさない

## 含める機能（MVPスコープ）

| # | 機能 | 検証仮説との対応(→02) |
|---|---|---|
| F1 | ペット登録（名前・犬猫・年齢任意） | 前提機能 |
| F2 | 薬の登録（薬名・用量自由文=獣医指示のまま・時刻スロット・曜日・期間・メモ） | 前提機能 |
| F3 | 投薬記録: 投薬済み / 見送り / 遅れて投薬（+訂正=取消レコード） | H1, H2 |
| F4 | 「誰が・いつ・何を」のappend-only監査履歴 | H1 |
| F5 | 家族・世話担当者の招待（6桁コード、72h期限） | H3 |
| F6 | 複数端末間のリアルタイム共有（Firestore） | H1, H3 |
| F7 | 二重記録・二重投薬の警告（スロット単位） | H1 |
| F8 | 投薬リマインダー通知（正確なアラーム、拒否時±10分に劣化）+ 通知から1タップ記録 | H2 |
| F9 | オフライン記録 + 通信回復後の自動同期（Firestoreオフラインキュー） | H2 |
| F10 | 最低限のバックアップ（クラウド同期がバックアップを兼ねる。エクスポートは有料表示のみ） | H4 |
| F11 | アカウント・データ削除（世帯退出 / オーナーは世帯全削除 + Auth削除） | 規約要件 |
| F12 | イベント計測（→07。PII・医療入力値は送信しない） | 全仮説 |
| F13 | 料金画面（無料/月400円/年3,900円の比較表示 + 購入意向計測。Play Billing非実装） | H4 |

排泄・食事・症状の簡易記録は投薬継続率検証に不可欠でないため **v1.1へ繰延べ**（データモデルだけ将来拡張可能に設計）。

## 含めない機能（原則実装しない）

AI診断 / 病名判定 / 投薬量推奨 / 飲み合わせ判定 / 医療助言 / 病院代替機能 /
写真日記 / ペットSNS / 散歩GPS / フード管理 / 用品販売 / 病院検索 / 保険管理 /
大規模健康分析 / チャット / 多数のカスタマイズ / Play Billing本実装（検証後） /
PDF・CSV出力の実装（有料機能として**表示のみ**、purchase_intent計測対象） / iOS・Web版・Wear OS

機能追加提案時は、どの検証仮説（02参照）に必要かを必ず明記すること。

## 無料/有料境界（価格仮説）

- 試用: 14日間は全機能（家族共有含む）
- 無料版: 1匹 / 1人 / 薬2件 / 履歴7日
- 有料版: 月400円 or 年3,900円 — 家族共有・履歴無制限・監査履歴・バックアップ・PDF/CSV出力・複数ペット
- MVPでは課金せず、価格表示と購入意向（purchase_intent イベント）のみ計測

## 非機能要件（検証MVPとしての水準）

- 通知: 予定時刻±15分の発火成功 ≥99.5%（権限許可端末、自己計測）
- 同期: オフライン記録が回復後に消失しないこと（データ消失ゼロが最重要 — DDレポートB-4）
- クラッシュフリーユーザー ≥99.7%（closed test中はログ目視で代替）
- 医療機器・診断を示唆するUI文言の禁止（06参照）
- Google Play: 健康アプリ申告 / Data Safety / アカウント削除導線（アプリ内+Web）に準拠

## 技術構成（決定済み・比較表は05に併記）

- Android: Kotlin / Jetpack Compose / Material 3 / 単一モジュール / mock・prod 2フレーバー / 手動DI
- バックエンド: Firebase Spark（匿名認証 + Firestore + rules）。Cloud Functions・Room・Hiltは使わない
- アラーム: AlarmManager主体 + DataStoreスロットキャッシュ + WorkManager日次保険
- LP: GitHub Pages（静的）+ GAS Webアプリ計測（Sheets蓄積）
